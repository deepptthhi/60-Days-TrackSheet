# Day 1 – Foundations of Computer Science


## What did I learn?

Today was my first step into Computer Science. Before watching this lecture, I thought programming was mostly about learning languages like Python or Java.

What I understood today is that programming is only a way to express a solution. The actual skill is learning how to think about a problem and solve it in a logical way.

I also got a basic idea of how computers represent data and why CS50 starts with Scratch instead of directly teaching a programming language.


## What new concepts did I understand?

- Computer Science is mainly about problem solving.
- Before writing code, it's better to understand the problem first.
- Breaking a big problem into smaller parts makes it easier to solve.
- An algorithm is simply a series of steps to solve a problem.
- Computers understand only binary (0's and 1's).
- Text, images, videos, and audio all get converted into binary inside a computer.
- Scratch helps beginners learn programming logic without worrying about syntax.
- Concepts like variables, loops, conditions, and functions are common in almost every programming language.
- Debugging isn't a mistake it's part of the development process.


## What computer/software engineering fundamentals did I learn today?

Today I understood that a computer takes an input, processes it using the CPU, temporarily stores the required data in RAM while the programs are running, and saves files permanently in the storage.

I also learned that when I open a website, my browser (client) sends a request to a server over the Internet. Instead of remembering an IP address, I can type a domain name like `google.com`, and DNS translates it into the correct IP address. The browser and server then communicate using HTTP or HTTPS.

Since I'm just starting out, I wanted to understand some basic terms that I'll keep seeing throughout my journey.


## Computer Fundamentals

- **Computer** → Takes an input, processes it, and gives an output.
- **Operating System (OS)** → Runs the computer and lets me use applications like Chrome or VS Code.
- **CPU** → Does all the processing and calculations.
- **RAM** → Temporarily stores data while applications are running.
- **Storage** → Keeps files and applications even after the computer is turned off.

## Internet Fundamentals

- **Internet** → Connects computers all over the world.
- **Network** → Devices connected so they can communicate.
- **Client** → Asks for data (my browser is a client).
- **Server** → Sends the requested data.
- **Browser** → Used to open websites.
- **Website** → A collection of web pages.
- **Web Page** → A single page inside a website.

## Web Basics

- **URL** → The address of a webpage.
- **Domain Name** → An easy to remember website name like `google.com`.
- **DNS** → Finds the IP address for a domain name.
- **IP Address** → The unique address of a device on a network.
- **HTTP / HTTPS** → Rules used by browsers and servers to communicate (HTTPS is secure).

Now I realize that languages are just tools. If I don't know how to think through a problem, knowing syntax alone won't help me build good software.
