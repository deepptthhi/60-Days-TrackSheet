# Day 15 – Revising Everything I Learned So Far

## What did I learn?

Today was dedicated entirely to revision.

Instead of starting a new topic, I wanted to spend the day revisiting everything I had learned over the past two weeks. Looking back at my notes and projects made me realize how much progress I've made since Day 1.

I started by revising the Python fundamentals I learned in the beginning of this journey. I went through variables, data types, loops, conditional statements, functions, strings, lists, tuples, dictionaries, sets, and object oriented programming. Revisiting these topics helped me connect concepts that had initially felt separate.

Next, I reviewed Git, GitHub, and Linux. I refreshed the complete Git workflow, common Linux commands, file permissions, process management, and system monitoring. It reminded me how important these tools are in day-to-day development.

After that, I revisited my **SmartExpiry** project. Reading through my own code helped me understand the project structure, class design, and overall program flow much better than when I first built it.

I then moved on to frontend development by revising HTML, CSS, and JavaScript. I reviewed semantic HTML, Flexbox, Grid, responsive design, DOM manipulation, events, arrays, objects, asynchronous JavaScript, and how JavaScript brings interactivity to web pages.

I also spent time looking through my **Local Utility Downtime Tracker** project. Instead of only reviewing the UI, I focused on understanding the logic behind each feature—how Local Storage manages data, how the weather API works, how Chart.js displays analytics, and how JavaScript ties everything together.

Finally, I revised both **React** and **TypeScript**. I reviewed JSX, components, props, state, event handling, conditional rendering, forms, lists, and hooks in React. Along with that, I refreshed TypeScript concepts like static typing, interfaces, type aliases, enums, generics, functions, classes, and how TypeScript helps catch errors before runtime while making large applications easier to maintain.

By the end of the day, I felt much more confident because I wasn't just remembering syntax, I was understanding why each technology is used and how everything fits together.

## Key Takeaways

- Revised Python fundamentals from basics to object oriented programming.
- Reviewed Git, GitHub, and Linux concepts.
- Revisited my SmartExpiry project and better understood its code structure.
- Strengthened my HTML, CSS, and JavaScript fundamentals.
- Reviewed my Local Utility Downtime Tracker project and the logic behind each feature.
- Reinforced React concepts including components, props, state, hooks, and rendering.
- Revised TypeScript fundamentals such as types, interfaces, generics, and classes.
- Connected everything I've learned into a much stronger overall understanding of web development.

## Reflection

Today's revision reminded me that learning isn't just about moving to the next topic, it's also about strengthening what I've already learned. Revisiting my notes and projects made me realize how much more comfortable I am with concepts that once felt confusing.

The biggest takeaway from today was seeing how all these technologies connect. Python strengthened my programming fundamentals, Git and Linux improved my development workflow, frontend technologies helped me build interactive applications, React introduced component based development, and TypeScript showed me the importance of writing safer and more maintainable code.
