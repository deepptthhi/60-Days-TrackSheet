# Day 13 - Learning React

## What did I learn?

Today I finally started learning **React**, something I've been looking forward to for quite some time.

At first, I wasn't sure what made React so different from HTML, CSS, and JavaScript. But as I went through the basics, I slowly started understanding why almost every modern web application uses it.

The biggest thing I learned today was that React is all about **thinking in components**. Instead of creating one large webpage, we break it down into small reusable pieces like a Navbar, Footer, Button, or Card. It felt like building with LEGO blocks instead of carving everything out of one giant piece.


Another new concept for me was **JSX**. In the beginning, seeing HTML inside JavaScript looked really confusing. I kept wondering, "Is this HTML or JavaScript?" But after spending some time with it, I realized JSX simply makes it easier to describe what the UI should look like.

As I continued learning, I also understood the basics of **props**, **components**, **state**, and **event handling**. I know I still have a long way to go before I become comfortable with React, but today gave me a good starting point.

Overall, today's learning felt less like memorizing syntax and more like learning a completely new way of building websites.


# What challenges did I face?

The hardest part today wasn't writing code it was changing the way I think.

Until now, whenever I wanted to create a webpage, I thought about HTML, CSS, and JavaScript as three separate things.

React completely changes that mindset.

Initially, I found JSX a little difficult because it combines JavaScript and HTML in one place. My brain kept expecting them to be separate, so it took some time to adjust.

I also noticed that React projects have many folders and files compared to a simple frontend project. Looking at them for the first time felt slightly overwhelming. There were moments when I wondered, "Do I really need all these files just to build a webpage?"

But as I explored them one by one, things slowly started making sense.

Another small challenge was remembering new terminology like components, props, JSX, and state. Since everything was new, I had to stop frequently and think about what each concept actually meant.



# What new concepts did I understand?

## Components

The idea of components was probably my favorite thing I learned today.

Instead of copying and pasting the same code multiple times, we can build a component once and reuse it anywhere in the application.

It made me realize why React applications are easier to maintain as they grow larger.


## JSX

Today I learned that JSX is simply a cleaner way of writing UI.

Although it looks like HTML, React converts it into JavaScript behind the scenes.

It felt unusual at first, but after writing a few examples, it actually started feeling more natural.



## Props

I learned that **props** are used to send information from one component to another.

I liked thinking of props as function arguments.

Instead of creating multiple similar components, we can simply pass different values and reuse the same component.

That made React feel much more powerful than plain HTML.


## State

I also got introduced to **state**.

What I understood today is that state stores information that changes while the application is running.

The best part is that React automatically updates the webpage whenever the state changes.

It almost felt like React was doing the difficult work for us.



## Events

I also learned how React handles events like button clicks.

Instead of manually selecting elements and adding event listeners like I did in JavaScript, React allows us to connect functions directly to components.

It made the code feel cleaner and more organized.


# What computer/software engineering fundamentals did I learn today?

Today's learning taught me an important software engineering principle: **building software in small, reusable pieces**.

Instead of writing one huge file, React encourages developers to divide the application into smaller components that each have a single responsibility.

I also realized why reusability is such a big part of software engineering.

Writing something once and using it multiple times not only saves time but also makes future updates much easier.

Another thing I understood is that modern web development is not just about making websites look good.

It's also about writing code that is organized, scalable, and easy for other developers to understand.


# What changed in my thinking?

Before today, I thought React was just another JavaScript library that people used because it was popular.

After spending time learning it, I realized React is actually a different way of thinking.

Instead of focusing on individual HTML pages, React teaches you to think about building reusable pieces that come together to form an entire application.

I also realized that learning React isn't something that will happen overnight.

There are many new concepts, and it will take practice before everything feels natural.

But surprisingly, that didn't discourage me.

Instead, it made me excited because I could already see how much easier building larger applications can become once I understand React properly.

Today felt like the beginning of another learning journey, just like when I first started learning JavaScript. I know there will be moments where I'll struggle, but I also know that every concept I understand now will make building future projects much easier.
