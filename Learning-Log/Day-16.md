# Day 16 - Learning Operating Systems

## What did I learn?

Today I started learning **Operating Systems**, and I wasn't expecting it to be this interesting.

Before today, I honestly never thought much about what an operating system actually does. I knew Windows, Linux, and Android were operating systems, but in my mind, they were just something that helped me use a computer.

As I started learning, I realized I had been taking it for granted all this time.

Something that surprised me today was how much work an operating system does without us ever noticing. Every time I open an application, save a file, connect my headphones, or even press a key on the keyboard, the operating system is involved in making sure everything happens properly.

I also learned that the operating system is like a bridge between the software we use and the hardware inside the computer. Until today, I never really wondered how an application gets memory, how it uses the CPU, or how it talks to the hardware. Learning about all of this made me realize that there is a lot happening behind the scenes.

I came across concepts like **kernel, shell, system calls, user mode, kernel mode, interrupts, booting, BIOS, UEFI, virtual machines**, and different types of operating systems. There were definitely a lot of new terms, and I won't say I understood everything perfectly on the first try. But I slowly started connecting the dots, and things began making much more sense.

By the end of the day, I wasn't just learning definitions anymore. I was actually starting to understand what my computer is doing every time I use it.

## What challenges did I face?

Today's biggest challenge was remembering all the new terminology.

There were so many words that I had never heard before, and at first they all sounded very similar.

For example, I kept mixing up the **kernel** and the **operating system** because I thought they were the same thing. It took me a while to understand that the kernel is actually just one part of the operating system, even though it's probably the most important one.

I was also a little confused about **user mode** and **kernel mode**. Initially, I didn't understand why applications couldn't directly access the hardware. But once I learned that it's done to protect the system from crashes and security issues, it finally made sense.

The booting process was another topic that surprised me. I always thought pressing the power button immediately started Windows, but today I learned there are several steps before the operating system even appears on the screen.

Although there were a lot of concepts, I found that trying to imagine what was happening inside the computer helped me understand them much better.

## What new concepts did I understand?

### Operating System

The biggest thing I understood today was what an operating system actually does.

Earlier, I thought it simply helped us use a computer.

Now I know that it's responsible for managing memory, processes, files, hardware devices, and so many other things that applications need to run properly.

### Kernel

The **kernel** was probably the concept that stood out the most.

At first, I thought it was just another technical word.

But after learning about it, I realized it's the part of the operating system that communicates directly with the hardware and manages everything happening inside the computer.

### System Calls

I also learned about **system calls**.

I found it interesting that applications don't directly interact with the hardware. Instead, they request the operating system whenever they need something like opening a file or accessing memory.

That made me understand why the operating system is always in control.

### User Mode and Kernel Mode

This was another concept that really changed my understanding.

Applications run in **user mode**, while the operating system runs in **kernel mode** with full access to the hardware.

I liked this idea because it makes the whole system much safer.

### Booting Process

I also learned what actually happens after pressing the power button.

I had never heard about BIOS, UEFI, bootloaders, or the kernel loading into memory before today.

Understanding those steps made me appreciate how much happens before I even see the login screen.

## What computer/software engineering fundamentals did I learn today?

One thing I realized today is that building software isn't just about writing code.

There has to be something that manages that code while it's running.

The operating system decides which program gets CPU time, where it should be stored in memory, how files are accessed, and how different applications work together without interfering with each other.

I also understood why operating systems are considered one of the most important subjects in computer science.

No matter what technology I learn in the future, everything will eventually depend on the operating system.

## What changed in my thinking?

Before today, I used my computer without ever thinking about what was happening behind the scenes.

If something opened, I just assumed it worked.

If a file got saved, I never wondered how.

Today's learning made me realize that the operating system is constantly making thousands of decisions every second, and we hardly ever notice it.

I also realized that learning operating systems feels different from learning programming languages.

When I learn Python or TypeScript, I'm learning how to build applications.

But today, I was learning what actually happens after those applications start running.

That was a completely different perspective, and I really enjoyed it.

I still have a lot to learn, and there were definitely topics that I'll need to revisit.

But compared to when I started this morning, I now look at my computer very differently.

Instead of just seeing applications on my screen, I now know there's an operating system quietly managing everything in the background.

I think that's probably the biggest thing I learned today.
