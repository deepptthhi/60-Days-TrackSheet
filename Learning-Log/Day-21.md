# Day 21 - Building the ClearView AI Accessibility Engineering Platform

## What did I learn?

Today felt less like studying and more like building something that could actually help people.

I worked on **ClearView**, an AI-powered Accessibility Engineering Platform that helps developers find accessibility issues in websites and documents. Before writing any code, I spent time thinking about the problem I wanted to solve, who would use the platform, and what features would make it useful. That made me realize that good projects start with planning, not coding.

After planning everything, I designed the structure of the project and started building the accessibility scanner. It can scan websites, HTML files, PDFs, and DOCX files to find accessibility issues. I also added AI so that instead of showing confusing technical errors, it explains each issue in simple language and suggests possible fixes.

I also worked on the dashboard to make the reports clean and easy to understand with accessibility scores, issue summaries, and recommendations. By the end of the day, I realized that building software involves much more than just writing code. It also includes planning, designing, testing, debugging, and continuously improving the product.


## What challenges did I face?

The biggest challenge today was deciding how everything should fit together.

I had many ideas, but I couldn't build everything at once. I had to decide which features were important for the first version and which could be added later.

I also spent a lot of time fixing bugs like filters not working correctly, scan results not updating properly, and making AI explanations simple enough for anyone to understand. Even though it was challenging, solving each problem helped me understand the project much better.



## What new concepts did I understand?

### Planning Before Coding

I learned that spending time planning a project before coding makes development much easier.

### Software Architecture

I understood how the frontend, backend, AI, and accessibility scanner work together as one complete application.

### Accessibility

I learned why accessibility is important and how issues like missing alt text, poor headings, and unlabeled forms affect users.

### AI Integration

I understood that AI can do more than generate content—it can also explain technical problems in a way that's easy for anyone to understand.

### Building Better User Experiences

I learned that presenting information clearly through dashboards, scores, and organized reports makes an application much more useful.

## What computer/software engineering fundamentals did I learn today?

Today's project showed me that software engineering is much more than coding.

It starts with understanding the problem, planning the solution, designing the application, building features step by step, testing everything, fixing bugs, and improving the product over time.

I also learned the importance of keeping different parts of a project separate so they are easier to maintain and expand in the future.


## What changed in my thinking?

Before today, I thought completing a project simply meant finishing the code.

Now I understand that coding is only one part of the process. Planning, designing, testing, debugging, and improving the user experience are just as important.

Working on ClearView also made me realize how important accessibility is. Small improvements can make a huge difference for people using a website.

Seeing this project slowly come together gave me a lot more confidence because I was finally applying everything I'd been learning into one real project.
