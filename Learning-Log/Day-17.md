# Day 17 - Starting with Node.js

## What did I learn?

Today I finally started learning **Node.js**, and honestly, I was pretty excited because this is where backend development actually begins.

For the last few weeks, I've been learning frontend development. I built web pages, made them interactive, learned React, and recently finished TypeScript. But somewhere in the back of my mind, I always had this question: *"Okay... but what happens after I click the Login button?"*

I knew there had to be something happening in the background, but I had no idea how it actually worked.

Today's learning answered a lot of those questions.

The first thing I found out was that **Node.js isn't a programming language**, and I actually laughed a little because that's what I believed until today. I thought backend meant learning an entirely new language, but it turns out I can continue using JavaScript.

That honestly made me feel relieved.

One thing that really caught my attention was that Node.js lets JavaScript run outside the browser. Until today, JavaScript only existed inside Chrome for me. I never imagined the same language could be used to create servers and handle requests from users.

As I kept learning, I slowly started understanding why Node.js became so popular.

I learned about **npm**, **package.json**, built-in modules, creating my first server, request and response, asynchronous programming, and something called the **Event Loop**.

I'm definitely not going to say I mastered all of these today.

Some concepts made sense immediately.

Some took a couple of examples.

And a few of them, especially the Event Loop, are things I'll probably understand much better once I start building projects.

One of my favorite moments today was when I created my first Node.js server.

It was literally just a "Hello World" server.

Nothing fancy.

But seeing JavaScript respond from the backend instead of changing something on a webpage felt strangely satisfying.

It almost felt like I had unlocked another side of JavaScript that I didn't even know existed.

By the end of today's learning, I wasn't just looking at Node.js as another technology to learn.

I started looking at it as the piece that connects everything I've learned so far.

## What challenges did I face?

Today's biggest challenge wasn't writing code.

It was changing the way I thought about JavaScript.

For the longest time, JavaScript only meant frontend to me.

Whenever someone said JavaScript, I immediately thought about buttons, forms, and websites.

Today I had to completely change that mindset and accept that JavaScript can also run on servers.

That took a little time.

Another thing I struggled with was understanding asynchronous programming.

Initially, I thought a server simply handled one request, finished it, and then moved to the next one.

Learning that Node.js can continue handling other requests while one task is still running was something I had to read more than once before it finally clicked.

The Event Loop was probably the most confusing topic today.

I understand the basic idea now, but I know this is one of those concepts that will become clearer only after I use it in real projects.

## What new concepts did I understand?

### Node.js

The biggest thing I understood today was what Node.js actually is.

Earlier, I genuinely believed it was another programming language.

Now I know it's a runtime environment that allows JavaScript to run outside the browser, making backend development possible with the same language.

### npm

I also learned about npm.

At first it looked like just another command.

But after using it, I understood why everyone talks about it.

Instead of building everything from scratch, we can install packages created by other developers and use them in our own projects.

### package.json

I also understood why every Node.js project has a package.json file.

Initially it looked like another file that got created automatically.

Now I know it's one of the most important files in the project because it keeps track of dependencies, scripts, and other important project information.

### Creating My First Server

This was probably my favorite part of today's learning.

Creating a server using just a few lines of JavaScript made backend development feel much less complicated than I had imagined.

Seeing **"Hello World"** appear in the browser because of my own server felt really satisfying.

### Request and Response

Another concept that made sense today was how communication happens between a browser and a server.

Whenever we open a website, the browser sends a request.

The server receives that request, processes it, and sends back a response.

It sounds simple, but understanding this made me realize that almost everything we do on a website follows this pattern.

### Event Loop

The Event Loop was probably the most interesting thing I came across today.

Initially I couldn't understand how Node.js could be single-threaded and still handle thousands of users.

After spending some time reading about it, I understood that Node.js doesn't simply sit and wait for long-running tasks.

Instead, it keeps handling other requests and comes back once those tasks are completed.

I still have a lot to learn here, but today I at least understood why the Event Loop is considered one of the most important concepts in Node.js.

## What computer/software engineering fundamentals did I learn today?

Today's learning made me realize that backend development is happening all around us, even though we never actually see it.

Every login.

I also realized that writing backend code isn't just about storing data.

It's about receiving requests, processing them efficiently, and responding as quickly as possible.

That gave me a completely different perspective on how web applications actually work.
