# Day 9 – Building My Foundation for DSA, Frontend (HTML, CSS) & Linux

## What did I learn?

Today was one of those days where I focused more on strengthening my basics than learning something completely new.

I started with **Arrays**, which is the first topic in DSA. Instead of jumping into LeetCode problems, I wanted to understand the fundamentals properly. I learned what arrays are, how they are stored in memory, why indexing starts from zero, and why some operations like accessing an element are fast while insertion and deletion take more time. I also went through basic operations like traversing, searching, updating, inserting, deleting, and their time complexities.

After that, I revised the **basics of HTML and CSS**. It had been a while since I looked at the fundamentals, so it was a good refresher. I revised HTML tags, forms, tables, semantic elements, and CSS concepts like selectors, the box model, Flexbox, positioning, and responsive layouts. It reminded me that even though these topics seem simple, they're the foundation for every frontend project.

I also spent some time learning **Linux basics**. I practiced using the terminal, moving between folders, creating and deleting files, and understanding the Linux file structure. At first, using only commands felt a little strange, but after some practice, it started to feel more comfortable.

I didn't just spend the day learning theory. I also worked on my **Local Utility Downtime Tracker** project. I cleaned up the folder structure, organized the files better, made a few UI improvements, and fixed some small issues that I had noticed earlier. The project feels much cleaner now, even though I didn't add any new features.

Looking back, today wasn't about finishing a big project or solving lots of problems. It was more about building a stronger base that will help me in the coming weeks.

## What challenges did I face?

The hardest part today was remembering the time complexity of different array operations. I knew what each operation did, but understanding *why* accessing an element is **O(1)** while insertion and deletion are usually **O(n)** took some time. Once I understood how arrays are stored in memory, it finally clicked.

While revising HTML, CSS, and Linux, I also realized I had forgotten a few basics. It was tempting to skip them, but going through them again helped refresh my understanding.

Another small challenge was organizing my project properly. I realized that keeping the folder structure clean is just as important as writing the code because it makes the project much easier to work on later.

## What new concepts did I understand?

Today I understood that learning DSA isn't just about solving problems it's about understanding how data is stored and why different approaches have different efficiencies.

I also got more comfortable with Linux and realized that using the terminal is something every developer should know because it's used almost everywhere.

Another thing I learned was that improving a project isn't only about adding features. Cleaning the code, organizing files, and making small improvements also make a big difference.

## What computer/software engineering fundamentals did I learn today?

One thing I realized today is that strong fundamentals make everything else easier.

Whether it's DSA, frontend development, or Linux, every advanced topic depends on understanding the basics first.

I also learned that software development is much more than writing code. Keeping projects organized, maintaining clean files, and continuously improving them are all part of becoming a better developer.

## What changed in my thinking?

Today made me realize that I don't always have to chase the next big topic.

Sometimes slowing down, revising what I already know, and understanding the "why" behind things is more valuable than trying to learn something new every day.

I'm starting to enjoy the process of building my foundation instead of worrying about how quickly I can finish my roadmap.
