# Day 10 – Learning More About Linux

## What did I learn?

Today I continued learning Linux, but instead of focusing on basic commands, I spent time understanding how Linux actually manages a computer.

The first thing I learned was **file permissions**. Earlier, I knew I could create and edit files, but I never really thought about who is allowed to access them. Today I understood that Linux gives different permissions to different users, which is one of the reasons it is considered a secure operating system. I practiced changing permissions and ownership, and it finally started making sense.

After that, I learned about **process management**. Before today, I had never really thought about what happens after opening an application. I learned that every program running on a computer is called a process, and Linux keeps track of all of them. I also explored a few commands that let me see which programs are currently running and how they can be managed.

I then moved on to **system monitoring**. It was interesting to see that Linux provides commands to check things like CPU usage, memory usage, system uptime, and running processes. It made me realize that developers don't always need a graphical application to understand what's happening inside a system.

Later, I spent some time learning **networking** commands. I learned how to check my system's IP address and verify whether the network connection is working properly. I haven't gone deep into networking yet, but this gave me a basic understanding of how Linux systems communicate.

Finally, I learned about **disk management**. I practiced checking the available storage space and understanding how Linux reports disk usage. These commands seemed simple, but I can already see how useful they'll be while working on larger projects or servers.

Overall, today's learning gave me a better picture of what happens behind the scenes in a Linux system.

## What challenges did I face?

Today's topics were a little different from what I've been learning recently because they weren't about writing code. Instead, they were about understanding how the operating system works.

The topic that confused me the most was **file permissions**. At first, the combinations of `r`, `w`, and `x` looked like random letters. After trying a few examples, I finally understood that they simply represent **read**, **write**, and **execute** permissions.

I also found it difficult to remember all the different commands used for process management and monitoring. Instead of trying to memorize every command, I focused on understanding what each one does. I think they'll become easier as I use them more often.

Networking was another completely new area for me. Some commands looked unfamiliar, but I didn't worry too much because my goal today was only to understand the basics.

## What new concepts did I understand?

### File Permissions

Today I learned that Linux doesn't allow everyone to access every file in the same way.

Permissions decide who can read, modify, or execute a file. It made me realize how Linux protects files and why permissions are such an important part of system security.

### Process Management

I understood that every application running on a Linux system is treated as a process.

Learning how to view and manage these processes helped me understand how the operating system handles multiple tasks at the same time.

### System Monitoring

I learned that Linux provides several commands to monitor CPU usage, memory usage, uptime, and running processes.

These commands give a quick idea of how the system is performing without needing any additional software.

### Networking

Today I got introduced to a few networking commands.

Although they were basic, they helped me understand how to check network information and verify whether a system is connected properly.

### Disk Management

I also learned how to check storage usage and available disk space.

It seems like a small thing now, but I can understand why monitoring disk space is important, especially when working with servers.

## What computer/software engineering fundamentals did I learn today?

One thing I realized today is that software doesn't just run on its own. Every application depends on the operating system to manage files, memory, processes, networking, and storage.

I also understood that developers should know more than just programming. Having a basic understanding of how the operating system works makes it much easier to debug problems and work with real-world systems.

## What changed in my thinking?

Until now, I mostly thought of Linux as a place where I type commands.

After today's learning, I started seeing it differently.

Every command I run is actually interacting with different parts of the operating system. Whether it's checking storage, viewing running processes, changing file permissions, or testing a network connection, Linux is constantly managing everything happening in the background.

It also made me realize that becoming a good software engineer isn't only about learning programming languages. Understanding the operating system is just as important because that's where our applications eventually run.
