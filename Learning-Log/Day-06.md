# Day 6 – Python Dictionaries, Sets & Functions

## What did I learn?

Today I learned about **dictionaries, sets, and functions** in Python.

I started with dictionaries and learned how they store data using **key-value pairs**. I practiced creating dictionaries, accessing values using keys, adding new data, updating existing values, removing items, and looping through them. I also explored useful methods like `get()`, `keys()`, `values()`, `items()`, `update()`, `copy()`, `pop()`, `popitem()`, `clear()`, and `setdefault()`. Towards the end, I learned about **dictionary comprehension** and **nested dictionaries**.

After that, I moved on to sets. The biggest thing I learned is that sets only store **unique values**, so duplicate values are removed automatically. I also practiced adding and removing elements, looping through sets, and performing operations like `union()`, `intersection()`, and `difference()`. I also learned what a **frozenset** is and how it is different from a normal set.

Finally, I learned about functions. I understood why functions are useful and how they help avoid repeating the same code. I practiced creating functions using `def`, passing values using parameters and arguments, using default and keyword arguments, and returning values using the `return` statement. I also got introduced to local and global variables and learned how variable scope works.

## What new concepts did I understand?

### Python Dictionaries

Before today, I thought dictionaries were just another way of storing data.

After writing a few programs, I understood why they are so useful. Instead of remembering the position of a value like in a list, I can simply use its key to access it. That makes the code much easier to read.

I also found `get()` really useful because it lets me access a value without worrying about getting an error if the key doesn't exist.

Dictionary comprehension looked confusing at first, but after trying a few examples, I realized it's just a shorter way of creating dictionaries.

### Python Sets

Sets were completely new to me.

The idea that a set automatically removes duplicate values made a lot of sense after trying a few examples. I also understood that sets don't maintain any order, which is why indexing doesn't work.

The set operations like `union()` and `intersection()` were interesting because they reminded me of what I learned in mathematics, but now I know how to use them in Python programs.

### Python Functions

Out of everything I learned today, functions probably made the biggest difference.

Earlier, whenever I wanted to repeat something, I would just write the same code again. Now I understand that it's much better to write it once inside a function and call it whenever I need it.

I also finally understood the difference between **parameters** and **arguments**, which used to confuse me. Learning about return cleared up something that used to confuse me. I finally understood that a function doesn't have to just print a value it can return it so I can use it somewhere else in my program.

## What computer/software engineering fundamentals did I learn today?

Today I realized that writing a program isn't just about getting the correct output. Choosing the right data structure and organizing the code properly also makes a big difference.

Dictionaries, sets, and lists all store data, but they solve different kinds of problems. Functions also showed me that breaking a program into smaller pieces makes the code easier to understand, test, and maintain.

Right now these topics feel basic, but I can already see that I'll keep using them in almost every Python program.

## What changed in my thinking?

Before today, I used to think that as long as my code worked, it was good enough.

After learning dictionaries, sets, and functions, I started thinking more about writing cleaner and more organized code. I realized that using the right data structure can make a solution much simpler.

Functions also changed the way I think about programming. Instead of copying the same code multiple times, I can write it once and reuse it whenever I need it. It also makes the code look much cleaner, and if I need to change something later, I only have to change it in one place.

Today's topics made Python feel much more practical, and I feel a little more confident than I did yesterday.
