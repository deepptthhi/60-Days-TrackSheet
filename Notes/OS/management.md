# Memory Management, Virtual Memory & File Systems

# 1. Why Memory Management?

Memory management allows many programs to run safely and efficiently.

Without it: - Programs overwrite each other's data. - RAM is wasted. -
System becomes unstable.

Real-life example: Apartments in a building. Every family gets its own
flat instead of everyone sharing one room.

# 2. Memory Hierarchy

Fastest to Slowest

Registers → Cache → RAM → SSD/HDD → Cloud/Backup

Remember: Closer to CPU = Faster and Smaller.

# 3. Address Binding

Address binding maps program addresses to actual memory.

Types: - Compile Time - Load Time - Execution Time

Execution-time binding allows relocation while the program is running.

# 4. Logical vs Physical Address

  Logical Address    Physical Address
  ------------------ ---------------------
  Generated by CPU   Actual RAM location
  Virtual            Real

Think of a hotel room number (logical) versus the physical room itself.

# 5. Memory Management Unit (MMU)

The MMU converts logical addresses into physical addresses.

CPU → MMU → RAM

The application never knows the real RAM address.

# 6. Contiguous Memory Allocation

A process receives one continuous block of memory.

Advantages: - Simple - Fast

Disadvantages: - Fragmentation

# 7. Fragmentation

## Internal Fragmentation

Unused space inside an allocated block.

## External Fragmentation

Free memory exists but is scattered into small pieces.

Memory Trick: Internal = Inside External = Outside

# 8. Swapping

The OS temporarily moves inactive processes from RAM to disk.

When needed, they are loaded back.

Real-life example: Moving old books from your desk to a shelf.

# 9. Paging

Paging divides: - Physical memory into Frames - Logical memory into
Pages

Pages can be stored in any free frame.

Advantages: - Eliminates external fragmentation. - Easy allocation.

# 10. Segmentation

Memory is divided according to logical parts: - Code - Data - Stack -
Heap

Unlike paging, segment sizes are different.

# 11. Paging vs Segmentation

  Paging                           Segmentation
  -------------------------------- ---------------------------
  Fixed-size blocks                Variable-size blocks
  Physical view                    Logical view
  Removes external fragmentation   Matches program structure

Remember: Paging = Equal Segmentation = Logical

# 12. Translation Lookaside Buffer (TLB)

TLB is a small cache storing recent page-table entries.

Benefit: Address translation becomes much faster.

Real-life example: Bookmarking frequently used pages in a book.

# 13. Virtual Memory

Virtual memory allows programs larger than RAM to execute.

It uses disk space as an extension of RAM.

Remember: Virtual Memory = Backup RAM

# 14. Demand Paging

Pages are loaded only when required.

Benefits: - Faster startup - Lower RAM usage

# 15. Page Fault

Occurs when a required page is not in RAM.

Steps: 1. CPU requests page. 2. Page not found. 3. OS loads page from
disk. 4. Program resumes.

# 16. Thrashing

Thrashing occurs when the system spends more time swapping pages than
executing programs.

Symptoms: - Slow computer - High disk usage - Low CPU utilization

Solution: Reduce multiprogramming or add RAM.

# 17. Page Replacement Algorithms

## FIFO

Removes the oldest page.

Simple but may suffer Belady's Anomaly.

## Optimal

Removes the page used farthest in the future.

Best performance but impossible to implement perfectly.

## LRU

Removes the least recently used page.

Widely used.

## Second Chance (Clock)

Improves FIFO using a reference bit.

# 18. File Systems

A file system organizes and stores files on storage devices.

Operations: - Create - Read - Write - Rename - Delete

# 19. Directory Structures

Types: - Single-Level - Two-Level - Tree - Acyclic Graph - General Graph

Tree structure is the most common.

# 20. File Allocation Methods

## Contiguous Allocation

Files stored together.

Fast access but difficult to expand.

## Linked Allocation

Blocks linked together.

Easy growth but slower random access.

## Indexed Allocation

Uses an index block pointing to all file blocks.

Used in many modern systems.

# 21. Free Space Management

Methods: - Bit Map - Linked List - Grouping - Counting

Purpose: Track unused disk blocks.

# 22. FAT vs NTFS vs ext4

  FAT           NTFS         ext4
  ------------- ------------ ------------
  Older         Windows      Linux
  Simple        Secure       Fast
  Small disks   Journaling   Journaling

# 23. Inodes & Journaling

## Inode

Stores metadata: - Owner - Permissions - Size - Location

Not the filename.

## Journaling

Records pending changes before writing them.

Benefit: Recovery after crashes.

# 24. Quick Revision

-   MMU = Address translator
-   Paging = Fixed pages
-   Segmentation = Logical divisions
-   TLB = Address cache
-   Virtual Memory = Disk as RAM
-   Page Fault = Missing page
-   Thrashing = Too much swapping
-   FIFO = Oldest page removed
-   LRU = Least recently used removed
-   Inode = File metadata
