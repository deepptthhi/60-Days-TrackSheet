

## What is Linux?

When I first heard about Linux, I thought it was just another operating system like Windows. After learning a little more, I understood that Linux is an operating system that helps us interact with the computer and manage its hardware and software.

The best part about Linux is that it is **open source**. This means anyone can look at its source code, improve it, and even create their own version. Because of this, Linux has become one of the most trusted operating systems in the world.

Today, Linux is used in servers, cloud platforms, software development, DevOps, cybersecurity, and many other technologies.

## Why should I learn Linux?

At first, I wondered why developers always talk about Linux. Then I realized that most servers and cloud systems run on Linux. Many companies also expect developers to know basic Linux commands.

Learning Linux makes it easier to work with servers, write code, use Git, deploy applications, and understand how computers work behind the scenes.

## What makes Linux special?

Here are a few things that make Linux different:

* It is free to use.
* Anyone can improve its source code because it is open source.
* It is fast and stable.
* It is considered very secure.
* It works well even on older computers.
* It allows multiple users to use the same system.
* It can run many programs at the same time.

## Linux vs Windows

Both Linux and Windows are operating systems, but they are used differently.

Windows is mainly used on personal computers and focuses more on graphical interfaces.

Linux is widely used on servers and by developers because it gives more control through the command line.

Neither is better than the other—it depends on what we want to do. For software development and cloud technologies, Linux is used much more often.

## What is the Kernel?

The kernel is the most important part of Linux.

I like to think of it as the **brain of the operating system**. Whenever a program wants to use the CPU, memory, keyboard, or any hardware, it first communicates with the kernel.

Without the kernel, Linux cannot work.

## What is the Shell?

The shell is the program that understands the commands we type.

For example, when I type a command like `pwd` or `ls`, the shell reads it, sends the request to the kernel, and then shows me the result.

In simple words:

**User → Shell → Kernel → Hardware**

The shell that is used most often is called **Bash**.

## What is the Terminal?

The terminal is simply the window where we type Linux commands.

Instead of clicking buttons, we can directly tell the computer what we want it to do by typing commands.

For example:

```bash
pwd
```

This shows the folder I'm currently working in.

```bash
date
```

This displays the current date and time.

The more I practice using the terminal, the more comfortable Linux becomes.

## Linux Distributions

Linux doesn't have just one version.

Instead, there are many different versions called **distributions** or **distros**. They all use the same Linux kernel but come with different tools and designs.

Some popular distributions are:

* Ubuntu
* Debian
* Fedora
* CentOS
* Arch Linux
* Kali Linux

As a beginner, Ubuntu is usually the easiest choice.

## Basic Linux Architecture

Linux works in a simple flow.

```text
User
   ↓
Shell
   ↓
Kernel
   ↓
Hardware
```

* The **user** gives a command.
* The **shell** understands the command.
* The **kernel** talks to the hardware.
* The **hardware** performs the task and sends the result back.

## Some Basic Linux Commands

These are some of the first commands I learned:

| Command    | What it does                   |
| ---------- | ------------------------------ |
| `pwd`      | Shows the current folder       |
| `ls`       | Lists files and folders        |
| `cd`       | Moves to another folder        |
| `clear`    | Clears the terminal screen     |
| `whoami`   | Shows the current user         |
| `date`     | Displays today's date and time |
| `hostname` | Shows the computer name        |
| `echo`     | Prints text on the screen      |

Example:

```bash
echo Hello Linux
```

Output:

```text
Hello Linux
```

## What I understood after learning Linux

When I first started, Linux looked difficult because everything was command-based. But after practicing a few commands, I realized it isn't as hard as I thought.

The more commands I use, the more comfortable I become. Instead of memorizing everything, I think it's better to understand what each command does and practice using it regularly.

* Linux is an open-source operating system.

* The terminal is where we type those commands.
* Linux has many distributions like Ubuntu, Debian, Fedora, and Kali.
