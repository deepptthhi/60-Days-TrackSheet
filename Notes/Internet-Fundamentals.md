

# 1. What is the Internet?

The Internet is a global network that connects millions of devices like computers, mobile phones, and servers.

It allows these devices to communicate and share information with each other.

Whenever I open a website, send an email, watch a YouTube video, or use an online application, I'm using the Internet.

### Key Points

- Connects devices around the world.
- Allows devices to communicate.
- Makes websites and online services possible.


# 2. What are Protocols?

Protocols are a set of rules that devices follow while communicating.

Just like people need a common language to talk to each other, computers also need common rules.

Examples of protocols are:

- HTTP
- HTTPS
- TCP
- UDP
- DNS

### Key Points

- Protocols are communication rules.
- Different protocols have different jobs.
- Without protocols, devices cannot communicate properly.



# 3. What is a Network?

A network is a group of devices connected together so they can communicate.

Examples:

- Home Wi-Fi
- Office network
- College computer lab

The Internet is the largest network in the world.

### Key Points

- Connects multiple devices.
- Allows devices to share information.
- Can be small or very large.



# 4. Types of Networks

## LAN (Local Area Network)

Connects devices within a small area.

Examples:

- Home
- Office
- School


## MAN (Metropolitan Area Network)

Connects devices across a city.



## WAN (Wide Area Network)

Connects devices over large distances.

The Internet is the biggest example of a WAN.

### Key Points

- LAN → Small area
- MAN → City
- WAN → Large geographical area



# 5. Internet Service Provider (ISP)

An ISP is a company that provides Internet access.

Examples:

- Airtel
- Jio
- ACT
- BSNL

Without an ISP, I cannot connect to the Internet.

### Key Points

- Provides Internet connection.
- Connects my device to the Internet.


# 6. Client & Server

Whenever I open a website, two things are involved:

### Client

The client requests information.

Example:

- Chrome
- Firefox
- Edge


### Server

The server stores the website and sends the requested information back to the client.

### Simple Flow

Browser (Client)

↓

Request

↓

Server

↓

Response

↓

Browser displays webpage

### Key Points

- Client requests information.
- Server sends information.
- Every website follows this model.



# 7. Browser

A browser is software used to open websites.

Examples:

- Google Chrome
- Firefox
- Edge
- Safari

A browser:

- Opens websites.
- Sends requests.
- Receives responses.
- Displays webpages.

### Key Points

- Used to access websites.
- Communicates with servers.
- Displays webpages.


# 8. Website vs Web Page

## Website

A collection of related webpages.

Example:

github.com


## Web Page

A single page inside a website.

Examples:

- Home Page
- Login Page
- About Page

### Key Points

- Website = Collection of pages.
- Webpage = One page inside a website.


# 9. Website vs Web Application

### Website

Mainly provides information.

Examples:

- Wikipedia
- College website


### Web Application

Allows users to perform actions.

Examples:

- Gmail
- YouTube
- GitHub
- ChatGPT

### Key Points

- Website → Read information.
- Web Application → Perform tasks.



# 10. URL (Uniform Resource Locator)

A URL is the complete address of a webpage.

Example:

https://www.google.com/search

A URL tells the browser where the webpage is located.

### Key Points

- Every webpage has a URL.
- Contains protocol, domain name, and path.


# 11. Domain Name

A domain name is the name of a website.

Examples:

- google.com
- github.com
- youtube.com

It is easier to remember than an IP address.

### Key Points

- Easy for humans to remember.
- Connected to an IP address.


# 12. IP Address (Internet Protocol Address)

An IP address is the unique address of a device connected to a network.

Computers communicate using IP addresses.

### IPv4

Example:

192.168.1.1

### IPv6

Example:

2001:db8::1

### Public IP

Used on the Internet.

### Private IP

Used inside local networks like homes and offices.

### Key Points

- Every device has an IP address.
- Computers use IP addresses to communicate.


# 13. DNS (Domain Name System)

DNS converts a domain name into an IP address.

Example:

google.com

↓

DNS

↓

IP Address

↓

Google Server

Without DNS, I would have to remember IP addresses instead of website names.

### Key Points

- Converts domain names into IP addresses.
- Makes websites easier to access.


# 14. DNS Cache

DNS Cache temporarily stores recently used DNS records.

This helps websites load faster because the browser doesn't need to ask the DNS server every time.

### Key Points

- Stores recent DNS lookups.
- Improves loading speed.


# 15. HTTP & HTTPS

HTTP is used for communication between the browser and the server.

HTTPS is the secure version of HTTP.

HTTPS encrypts the communication to make it safer.

### Key Points

- HTTP → HyperText Transfer Protocol, Normal communication.
- HTTPS → HyperText Transfer Protocol Secure, Secure communication.


# 16. Why HTTPS is Important

HTTPS protects sensitive information like:

- Passwords
- Banking details
- Personal information

It encrypts the data before sending it over the Internet.

### Key Points

- Keeps communication secure.
- Protects sensitive information.


# 17. Request & Response

Whenever I open a website:

1. Browser sends a request.
2. Server processes it.
3. Server sends a response.
4. Browser displays the webpage.

### Key Points

- Browser sends requests.
- Server sends responses.


# 18. TCP & UDP

## TCP

Transmission Control Protocol

Reliable communication.

Makes sure data reaches the destination correctly.


## UDP

User Datagram Protocol

Faster communication.

Doesn't guarantee delivery.

### Key Points

- TCP → Reliable.
- UDP → Fast.


# 19. Ports

Ports help different applications communicate over the network.

They act like different doors for different services.

### Key Points

- Used by applications for communication.


# 20. Common Ports

- HTTP → 80
- HTTPS → 443
- DNS → 53


# 21. Cookies

Cookies are small pieces of information stored in the browser.

They help websites remember users and preferences.


# 22. Sessions

Sessions help the server remember a user while they're using a website.

For example, they keep me logged in as I move between pages.

# 23. Cache

Cache temporarily stores frequently used data.

This helps websites load faster.


# 24. CDN (Content Delivery Network)

A CDN stores copies of website content on servers in different locations.

Users receive data from the nearest server, making websites load faster.


# 25. What Happens When I Type google.com?

1. I type google.com.
2. Browser asks DNS for the IP address.
3. DNS returns the IP address.
4. Browser connects to Google's server.
5. Browser sends an HTTPS request.
6. Server processes the request.
7. Server sends a response.
8. Browser displays the webpage.
